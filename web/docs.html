Documentation

Welcome to the documentation page! This page can be access without logging in by simply using /docs in the URL.

On this page, you'll learn about this application.

-- Let's begin --

Chapter 1:

To use this application you must first acknowledge the details displayed by the warning message shown before accesing any other page

of this application. This warning is not shown when accesing this (documentation) page. Please go to the root URL to view it.

After you followed the instructions given by the warning message, you will no longer see it until you restart this application.

The page which serves the message (/new_client) will no longer be accessible, instead, you will be redirected to the login page.

Keep in mind that this page is not is not supposed to be accessed by the user. This application should redirect the user to it internally.

Chapter 2:

On the login page, please enter your credentials. These can be defined in the source code of this application. If you entered the

correct credentials, you'll be given access to the management page (more about it later). The authentication is done by

calling the /auth URL. Keep in mind that this API is not supposed to be used by the user. It should be called

by this application itself internally. If the warning message (mentioned in Chapter 1) has not been acknowledged, the authentication

request will be invalidated even if the login credentials are correct. Same behaviour applies if the user is already authorized/logged in.

Chapter 3:

The user management page (/man) allows you to see all the people stored in the database and manage their data. Keep in mind that both

the usernames and passwords are limited to 16 characters in the HTML code. This page can only be accessed after acknowledging the

warning message (mentioned in Chapter 1) and logging in with valid credentials.

You can perform the following operations:

  • Add a new entry
  • Remove an entry
  • Edit the person's information (name, age, e-mail, etc.)

Please keep in mind that this application does not support user permissions. This means that any user who can log in, will be

able to perform any of the operations listed above.

Also, the database is regenerated on every start, so any changes you do will only apply until restart!

The End.

For now, this is all you need to know about this application.
